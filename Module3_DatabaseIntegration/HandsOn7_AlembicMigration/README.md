# Hands-On 7: Migrations & Versioning — Alembic and Django Migrations

**Course**: Digital Nurture 5.0 - Python Full Stack Engineer Track  
**Module**: Database Schema Version Control, Alembic Migrations & Safe Rollbacks  

---

## 📋 Overview

This hands-on exercise establishes database schema versioning using **Alembic** (for SQLAlchemy) and documents Django migration mechanisms. It tracks schema evolution across 3 incremental revisions:
1. **Revision 001**: Baseline schema initialization (5 core tables).
2. **Revision 002**: Incremental column addition (`is_active BOOLEAN DEFAULT TRUE` on `students`).
3. **Revision 003**: New table addition (`course_schedules` with Foreign Keys).

It demonstrates safe rollback strategies (`alembic downgrade -1`, `alembic downgrade base`, `alembic upgrade head`) and tracks revision state in the `alembic_version` database table.

---

## 📁 Files Included

- `alembic.ini`: Alembic runtime configuration file.
- `alembic/env.py`: Migration environment script importing `Base.metadata`.
- `alembic/versions/`: Contains autogenerated Python migration revision scripts (`001`, `002`, `003`).
- `models.py`: Evolving SQLAlchemy ORM models.
- `migration_commands.md`: Command cheat sheet for Alembic & Django migrations.
- `README.md`: Exercise documentation.

## 🚀 Migration & Rollback Commands

### 1. View Current Applied Revision
```bash
cd HandsOn7_AlembicMigration
alembic current
```

### 2. View Migration History Chain
```bash
alembic history --verbose
```

### 3. Apply All Pending Migrations to Head
```bash
alembic upgrade head
```

### 4. Roll Back One Step (Downgrade -1)
```bash
alembic downgrade -1
```

### 5. Roll Back All Migrations to Base
```bash
alembic downgrade base
```
